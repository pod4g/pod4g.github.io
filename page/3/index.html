{}的用法及对象字面量里的this

发表于 2016-04-09   |  

{} 如果不是作为赋值语句的话，就表示一个代码块

注意：在console中测试代码块语法永远会报错，因为虚拟机在执行时，会自作主张第地用分组符包裹着这个语句块，所以这个表达式肯定是返回一个对象的。（这种自作主张很好！绝大部分情况下是符合我们的预期的，究竟谁会无聊到在console中测试代码块语法呢？）

{} 如果作为赋值语句的话，就表示一个对象字面量

如果 {} 作为一个对象字面量的话，内部的语句只能是 key:value, 不能出现其他符号，否则报错 Uncaught SyntaxError: Unexpected token x。这就是使用eval来解析json字符串时，需要加上()的原因，不加的话，会把{}当作代码块，而代码块中不能出现 key:value, 这种代码

如果想要作为一个对象且可以在内部使用所有语句，可以参考 function 技巧中的 new function(){} 的使用

闭包总结

  1. 只要是函数嵌套函数都是闭包

  2. 闭包分为2种

    • 有意义的闭包，内部函数在外部函数外部执行（一般是return出去）,只有有意义的闭包，GC才不会回收
    • 无意义的闭包，内部函数在外部函数内部执行或return出去但是没有接收，此时这种闭包没有意义，GC照样会回收外部函数和内部函数的变量对象

闭包和柯里化

发表于 2016-03-12   |  

闭包和柯里化

JavaScript中只有两种执行环境，一种是全局执行环境，一种是局部执行环境

  • 全局执行环境：是最外围的一个环境，根据ECMAScript实现所在的宿主环境的不同，表示执行环境的对象也不一样，为WEB浏览器中，全局
    执行环境是window对象，因此所有全局变量和函数都是作为window对象的属性和方法创建的。当JS开始执行时，会构建一个全局
    执行环境，全局执行环境直到应用程序退出–例如浏览器关闭–时才会被销毁。

  • 局部执行环境：每当函数被调用的时候，就会产生一个新的环境，此环境就会被推入一个环境栈中，而在函数执行之后，栈将其弹出，把控制权返回给之前的环境。局部执行环境是一种概念，是运行中的函数的意思，他不是对象。

  • 某个执行环境执行完后，该环境被销毁，保存在其中的所有变量和函数定义也随之销毁

闭包和柯里化之间的关系

函数嵌套函数的形式叫做闭包，闭包的本质是 内部函数迟于外部函数执行

闭包分为2种：

1.有意义的闭包

  • return function 这种就是柯里化
  • return { xx:function(){} }
  • setTimeout/setInterval
  • 把内部函数的值赋值给一个不属于外部函数作用域之内的变量
  • 事件
  • …

2.无意义的闭包

内部函数在外部函数内部执行。即虽然是函数嵌套函数的形式，但其实内部函数是和外部函数一起执行的。

柯里化是闭包的一种，柯里化一定是闭包。但闭包不一定是柯里化（还有可能是其他形式）。
