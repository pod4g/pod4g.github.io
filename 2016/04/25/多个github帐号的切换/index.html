多个github帐号的切换

发表于 2016-04-25   |  

背景

我有两个Gighub账户，一个是 MyNameIsJimGreen 一个是 pod4g ，一个帐号是学习、私人项目使用，一个是工作使用。

但是现在有一个问题，如何在一个电脑上同时使用这两个账户呢？

对于Gighub来说，两个帐号是不能添加同一个SSH公钥的（Key is already in use）。

两个帐号也不能添加同一个email。

每个github帐号的ssh key并不一定非要使用注册时（绑定）的邮箱

MyNameIsJimGreen对应的邮箱是：455322185@qq.com
pod4g对应的邮箱是 ：1601942611@qq.com

我想默认使用pod4g账户

操作系统：Mac OSX EI Capitan 10.11.4
terminal：iTerm2
相关文章：http://blog.csdn.net/itmyhome1990/article/details/42643233

下面开始配置多个gihub帐号在同一台机器上使用

1.首先生成pod4g的ssh key

2.把生成的位于 ~/.ssh/ 下的 id_rsa.pub 公钥文件内的内容添加到pod4g的帐号中（在github网站操作）

3.验证是否验证成功

到这一步，pod4g的配置已经完成了

4.生成 MyNameIsJimGreen的ssh key

5.生成key时，不要使用默认名称，如果使用默认的话，会覆盖掉pod4g的key pairs
假如使用的key名称为 jim_rsa

6.在 ~/.ssh/ 下，vim config
内容为：

7.把生成的位于 ~/.ssh/ 下的 jim_rsa.pub 公钥文件内的内容添加到MyNameIsJimGreen的帐号中（在github网站操作）

8.验证是否验证成功

此时，在一台电脑上我就可以使用两个github帐号了。。

不过，在第一次使用时，

  • 给MyNameIsJimGreen的管理的项目指定远程仓库（已经存在的项目也需要这么做） git remote add origin_jim git@github_jim:MyNameIsJimGreen/Vuri-BoyKa.git

  • 把本地的需要MyNameIsJimGreen的管理的项目push到上一步指定的远程仓库中（在push之前可能需要add和commit） git push -u origin_jim master

以后使用的时候，就按照以前使用github的方式来使用即可。。
