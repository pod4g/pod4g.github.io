ES规范转型之双等和全等

发表于 2016-06-26   |  

关于双等

在比较双等时，javascript使用的是抽象相等比较算法

算法描述：

比较运算x==y，其中x和y是值，产生true或者false。这样的比较按如下方式进行：

 1. 若Type(x)与Type(y)相等，则
    a. 若Type(x)为Undefined，则返回true。
    b. 若Type(x)为Null，则返回true。
    c. 若Type(x)为Number，则
       i.   若x为NaN，返回false。
       ii.  若y为NaN，返回false。
       iii. 若x与y为相等数值，则返回true。
       iv.  若x为+0，y为-0，返回true。
       v.   若x为-0，y为+0，返回true。
       vi.  返回false。
    d. 若Type(x)为String，则当x和y为完全相等的字符序列（长度相等且相同字符出现在同样位置）时返回true。否则，返回false。
    e. 若Type(x)为Boolean，则当x和y同为true或者同为false返回true。否则，返回false。
    f. 若Type(x)为Object，则当x和y引用同一个对象时返回true。否则返回false。
2. 若x为null且y为undefined，返回true。
3. 若x为undefined且y为null，返回true。
4. 若Type(x)为Number且Type(y)为String，返回comparison x == ToNumber(y)的结果。
5. 若Type(x)为String且Type(y)为Number，
6. 返回toNumber(x) == y的结果。
7. 若Type(x)为Boolean，返回比较ToNumber(x) == y的结果。
8. 若Type(y)为Boolean，返回比较 x == ToNumber(y)的结果。
8. 若Type(x)为Object且Type(y)为String或Number，返回比较ToPrimitive(x) == y的结果。
10. 返回false。

从规范的定义可知抽象相等比较算法有时候会递归调用。

例如true == “abc”：
true == “abc”，根据7，把true转为1，即
1 == “abc”，然后再根据4，比较 1 == ToNumber(“abc”)，即
1 == NaN，然后根据1.c.ii，最终返回false

例如true == “1”：
true == “1”，根据7，把true转为1，即
1 == “1”，然后在根据4，比较 1 == ToNumber(“1”)，即
1 == 1，然后根据1.c.iii，最终返回true

关于全等

在比较全等时，javascript使用的是严格相等比较算法

算法描述：

比较x===y，x和y为值，需要产出true或false。比较过程如下：

  1. 若Type(x)与Type(y)的结果不一致，返回false，否则
  2. 如果Type(x)为Undefined，返回true
  3. 如果Type(x)为Null，返回true
  4. 如果Type(x)为Number，则
    a. 如果x为NaN，返回false
    b. 如果y为NaN，返回false
    c. 如果x和y为同一个数字，返回true
    d. 如果x为+0，y为-0，返回true
    e. 如果x为-0，y为+0，返回true
    f. 返回false
    1. 如果Type(x)为String，如果x与y为完全相同的字符序列（相同长度和相同的字符对应相同的位置），返回true，否则，返回false
    2. 如果Type(x)为Boolean，如果x与y都为true或false，则返回true，否则，返回false
    3. 如果x和y引用同一个Obejct对象，返回true，否则，返回false

从算法描述上可知，全等是双等的一种情况，对应的是双等的1
