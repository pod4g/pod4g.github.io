list删除相同元素的问题

发表于 2013-03-16   |  

今天研究了一下list的删除元素的问题，发现了很多容易出现错误的点，记录下来，防止自己再犯同样的错误

代码如下：

List list = new ArrayList();
list.add(“北”);
list.add(“京”);
list.add(“京”);
list.add(“北”);
for (String string : list){
if(“北”.equals(string)){
list.remove(string);
}else{
continue;
}
}
我要删除所有“北”，原先以为这样可以很正常的删掉所有“北”，但是，运行的时候出现异常了：
Exception in thread “main” java.util.ConcurrentModificationException
at java.util.AbstractList$Itr.checkForComodification(AbstractList.java:372)
at java.util.AbstractList$Itr.next(AbstractList.java:343)
at day07.pro.TestThread.main(TestThread.java:16)

我就想，如果把foreach循环改成普通的for循环会不会在出现这个问题呢？

试了一下：
List list = new ArrayList();
list.add(“北”);
list.add(“京”);
list.add(“京”);
list.add(“北”);
for(int i=0;i<list.size();i++){
if(“京”.equals(list.get(i))){
list.remove(i);
}else{
continue;
}
}
运行结果正常，但是如果要remove “京”就不行了，只会把第一个“京”，即，索引为1位置的“京”删除，
这是为什么呢？上网查了一下，发现也有人出现这个问题，用普通for循环如果要删除相同的元素只能删掉不相邻的元素，所以用这种方法是不可取的

解决办法：
方式一：
for(int i=0;i iter = list.iterator();
while(iter.hasNext()){
String s = iter.next();
if(s.equals(“京”)){
iter.remove();
}
}
注：使用Iterator遍历，通常情况下，会出现java.util.ConcurrentModificationException，比如在
while(it.hasNext()){
String str = it.next();
if(“北”.equals(str)){
list.remove(str);
}
}
即会出现此异常

解决办法：
while(it.hasNext()){
String str = it.next();
if(“北”.equals(str)){
it.remove();
}
}
