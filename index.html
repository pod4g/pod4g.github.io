对DOM的修改永远都是异步的

例如下面的一个场景：

在页面上有2个script标签，第一个script标签内动态创建了一个script，去加载一个config.js文件，
第二个script标签内，需要用到动态加载的config

config.js

index.html

出现上面问题的原因在于：

对dom的修改永远都是异步的

也就是说要等到所有已经加载的脚本执行完毕，再去修改DOM，然后加载并执行新的脚本

要根治这个问题，可以才用成熟的模块加载其如RequireJS,SeaJS

写了一段代码进一步验证 对dom的修改永远都是异步的

原先的理解是：js修改dom是同步的，执行很快，但是渲染线程跟js引擎线程是互斥的，由于下面还有js代码要执行
所以渲染线程得不到机会来渲染新的修改后的dom。

现在的理解：js修改dom是异步的，下面的代码执行完毕，然后再来更新dom树，这些都完毕之后，渲染线程才得到机会来渲染新的修改后的dom

原先理解的问题：没有考虑到有些dom是不需要渲染的，例如head内的标签。所以如果修改了一个不需要渲染的标签，其表现也是异步的，那么就可以得出结论对dom的修改永远都是异步的

  1. 核心模块（http/fs/path…）
  2. 文件（本地）模块（自定义的js文件）

linux中的压缩与解压缩

发表于 2016-05-28   |  

linux中常用的压缩格式

  • .zip
  • .gz
  • .bz2
  • .tar.gz
  • .tar.bz2

下列所有的压缩命令和打包命令，如果压缩文件和打包文件前面有路径，就压缩或打包到指定的路径下
如果路径不存在就报错。

压缩率：.tar.bz2 > .tar.gz > .zip

.zip格式

压缩命令为：zip 压缩文件名 源文件

例如：

  1. 压缩文件 zip test.zip test.txt
  2. 压缩目录 zip -r test.zip test/
  3. 压缩多个文件 zip test.zuo 1.txt 2.txt
  4. 压缩多个目录 zip -r test.zip d1 d2
  5. 压缩多个目录和文件 zip -r test.zip d1 test.txt

可操作所有目录下的文件，在前面加上绝对或相对路径即可

解压缩命令为：unzip 压缩文件名

例如：

  1. 普通解压命令 unzip test.zip
  2. 解压到指定目录 unzip test.zip -d /tmp/
  3. 解压到指定目录没有就创建一个 unzip test.zip -d /tmp/test
  4. 查看压缩包内容 unzip -v test.zip

.gz格式

压缩命令为：gzip 源文件

压缩为.gz格式的压缩文件，源文件会消失。

gzip -c 源文件 > 压缩文件
如： gzip -c test.txt > text.gz
压缩为.gz格式的压缩文件，源文件保留。
没有一个像bz2一样的选项可以保留源文件，只能用这种比较trick的方式

gzip -r 目录
压缩目录下所有的子文件，但是不能压缩目录本身。

解压缩命令为：

1.gzip -d 压缩文件

解压缩文件： gzip -d test.txt.gz
解压缩目录：gzip -dr test

2.gunzip 压缩文件

解压缩文件：gunzip 压缩文件
解压缩目录：gunzip -r 压缩目录

.bz2格式

压缩命令为：bzip2 源文件
压缩为.bz2格式的压缩文件，源文件会消失。

bzip2 -k 源文件
压缩为.bz2格式的压缩文件，源文件保留。

bzip2不能压缩目录

解压缩命令为：

1.bzip2 -d 压缩文件

解压缩，-k保留压缩文件

bzip2 -dk 压缩文件

2.bunzip2 压缩文件
解压缩，-k保留压缩文件

bunzip2 -k 压缩文件

tar打包

.gz不能压缩目录本身，.bz2参数压根不能跟目录，如果这样的话，那么这两个压缩命令就太有局限性了。

但是不要忘了，linux中有一个tar打包命令，可以先给目录打包，生成一个tar文件，然后在进行压缩

打包：tar -cvf 打包文件名 源文件

选项：
-c 打包
-v 显示过程
-f 指定打包后的文件名

例如： tar -cvf test.tar test

解打包：tar -xvf 打包文件名
解打包到指定目录：tar -xvf 打包文件名 -C /tmp

选项：
-x 解打包

例如： tar -xvf test.tar

tar命令不区分文件和目录

.tar.bz2

压缩命令：

tar -jcvf 压缩包名.tar.bz2 源文件

压缩多个文件或目录：

tar -jcvf 压缩包名.tar.bz2 源文件1 源文件2 目录

选项：
-j 压缩为.tar.bz2格式

解压缩命令：

tar -jxvf 压缩包名.tar.bz2

解压缩到指定目录：

tar -jxvf 压缩报名.tar.bz2 -C /tmp

选项：
-x 解压缩.tar.bz2格式

.tar.gz

压缩命令：

tar -zcvf 压缩包名.tar.gz 源文件

压缩多个文件或目录：

tar -zcvf 压缩包名.tar.gz 源文件1 源文件2 目录

选项：
-z 压缩为.tar.gz格式

解压缩命令：

tar -zxvf 压缩包名.tar.gz

解压缩到指定目录：
tar -zxvf 压缩报名.tar.gz -C /tmp

选项：
-x 解压缩.tar.gz格式

但是发现了一个奇怪的问题，就是在右侧数据操作区，更改了userData.score，左侧listView区中的user.score竟然也跟着发发生了变化。

这种情况并不是我想要的，我想要的是：listView区，只负责数据的展示。数据操作区负责数据的操作，且操作的是userData，保存时，也是保存的userData，即listView区中的user和操作区的userData是两个独立的VM，两者并不会绑定同一个view。

经过各种实验，终于找到了解除user和userData之间关联的办法：

后记：$scope.userData = user这句代码使得$scope.userData和user，指向了同一个object,
所以我在操作区更改$scope.userData.score，也会引起listView区的user.score的变化。

记住：在view中绑定事件时，传递的参数其实就是$scope上的同名属性指向的object，在事件回调里，并不会产生一个新的object。要是想要产生新的object就要使用深拷贝了。。
